# Verification Plan

Goal: define the evidence that would make a change trustworthy.

## Method

1. Identify the behavior that must be proven.
2. Choose the smallest relevant checks first.
3. Add broader checks when the change crosses boundaries or raises risk.
4. Name important edge cases, regressions, and manual review needs.
5. State what is intentionally not being verified and why.

## Output

- behavior to prove
- smallest relevant checks
- broader checks if warranted
- edge cases or regressions
- manual review needs
- known verification gaps
