# Change Planning

Goal: convert a request into a focused change plan before implementation.

## Method

1. Restate the intended outcome.
2. Identify the relevant project docs, modules, and likely touch points.
3. List assumptions and unknowns.
4. Propose the smallest coherent implementation path.
5. Name explicit non-goals to prevent scope creep.
6. Identify risks and what should be verified after the change.

## Output

- outcome
- relevant areas
- assumptions or unknowns
- implementation steps
- non-goals
- risks
- verification needs
